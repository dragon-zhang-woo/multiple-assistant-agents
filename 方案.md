# 期末作业方案：AI科研助手团队（Research Agent Team）

## 1. 实验背景与任务要求

本实验要求设计并实现一个多智能体系统 SmartAgents，核心是让大语言模型结合规划、记忆与工具完成复杂任务。根据实验 PDF，系统至少需要满足以下要求：

- 智能体数量不少于 4 个，每个智能体需要有清晰角色定位和提示词。
- 至少实现一种智能体推理/规划方法，例如 CoT、ReAct 或 Reflection。
- 至少设计 2 种记忆机制，包括短期记忆和长期记忆。
- 至少使用 2 种工具调用，例如 RAG 检索、搜索引擎、PDF 阅读、统计计算等。
- 提交时包含 `code/` 文件夹和实验报告 PDF，压缩包命名为“学号_姓名_实验七.zip”。

本项目选择“AI科研助手团队”作为期末作业场景。用户输入一个科研问题，例如“近年来 Agent Memory 有哪些研究方向？”，系统自动检索论文、阅读摘要和本地 PDF、提取主要贡献、反思局限，并生成调研报告与思维导图。

## 2. 系统总体设计

系统采用中心化流程编排，主路径使用 LangGraph 表示工作流：

```text
manager -> memory_retrieve -> search -> read -> critique -> write -> memory_update -> end
```

如果当前环境没有安装 LangGraph，程序会使用顺序执行兜底路径完成同样流程，并在 `outputs/run_log.json` 中记录 warning，保证课堂演示和无 API key 环境也能跑通。真实模型调用支持 DeepSeek 和 DashScope 两种 OpenAI-compatible provider。

核心状态统一保存在 `ResearchState` 中，包含：

- `topic`：用户输入的科研问题。
- `messages`：短期对话和推理轨迹。
- `tool_logs`：工具调用记录。
- `retrieved_memories`：长期记忆检索结果。
- `papers`：检索到的论文元数据。
- `paper_analyses`：Reading Agent 提取出的贡献、方法和局限。
- `stats`：统计工具输出。
- `report_path`、`mindmap_path`、`run_log_path`：最终产物路径。

## 3. Agent 设计

| Agent | 职责 | 方法 |
| --- | --- | --- |
| ManagerAgent | 拆解科研问题，制定调研计划，协调后续步骤 | Planning + CoT |
| SearchAgent | 使用 arXiv 检索论文，并记录 Thought/Action/Observe 过程 | ReAct |
| ReadingAgent | 阅读论文摘要和本地 PDF，提取贡献、方法、标签和初步局限 | 结构化信息抽取 |
| CriticAgent | 对论文和系统输出进行反思，指出不足和风险 | Reflection |
| WriterAgent | 汇总所有结果，生成 `survey.md` 和 `mindmap.md` | 综合写作 |

这种分工可以体现多 Agent 系统的优势：不同 Agent 具有不同职责，彼此通过共享状态协作，最终完成单个 Agent 较难稳定完成的科研调研任务。

## 4. 记忆机制设计

### 短期记忆

短期记忆保存在本轮运行的状态中：

- `messages` 保存系统消息、Agent 发言、规划结果和反思结果。
- `tool_logs` 保存每次工具调用的输入、输出数量、warning 和统计结果。
- `pdf_notes` 保存本地 PDF 的摘录和摘要。

短期记忆用于保证当前调研流程前后连贯，例如 WriterAgent 可以读取 SearchAgent 和 ReadingAgent 的结果生成报告。

### 长期记忆

长期记忆保存在 `data/long_term_memory.json` 中。每次运行结束后，系统会写入：

- 调研主题；
- 论文标题、年份和链接；
- ReadingAgent 提取出的标签；
- 报告路径；
- 运行时间。

下一次运行开始时，Memory 节点会用简单向量式 bag-of-words cosine similarity 检索历史主题和论文标签，把相似历史记录提供给后续 Agent。这样可以体现“长期记忆”和“历史经验复用”。

## 5. 工具调用设计

系统实现 3 类核心工具：

1. `arxiv_search`
   - 请求 arXiv Atom API 获取论文标题、作者、年份、摘要、链接和 PDF 链接。
   - 网络失败或 arXiv 无结果时，使用内置论文样例兜底，保证演示可运行。

2. `pdf_extract`
   - 使用 `pdfplumber` 读取本地 PDF 前几页文本。
   - 可通过命令行参数 `--pdf` 传入论文或实验材料。

3. `paper_stats`
   - 统计论文数量、年份分布和高频关键词。
   - 结果写入报告和运行日志。

此外，系统还包含 `memory_retrieve` 和 `memory_update` 两个记忆工具节点，用于体现长期记忆读写。

## 6. 运行方式

安装依赖：

```bash
pip install -r code/requirements.txt
```

无 API key 演示模式：

```bash
python code/main.py --topic "近年来Agent Memory有哪些研究方向？" --max-papers 5 --mock auto
```

带本地 PDF：

```bash
python code/main.py --topic "Agent Memory综述" --pdf "7-Agent-lab(1).pdf" --mock auto
```

DeepSeek 真实模型模式：

```bash
set DEEPSEEK_API_KEY=your_key_here
python code/main.py --provider deepseek --topic "近年来Agent Memory有哪些研究方向？" --max-papers 5 --mock never
```

DashScope/Qwen 真实模型模式：

```bash
set DASHSCOPE_API_KEY=your_key_here
python code/main.py --provider dashscope --topic "近年来Agent Memory有哪些研究方向？" --max-papers 5 --mock never
```

可选环境变量：

- `DEEPSEEK_API_KEY`：DeepSeek API 密钥。
- `DEEPSEEK_MODEL`：默认 `deepseek-v4-flash`。
- `DEEPSEEK_BASE_URL`：默认 `https://api.deepseek.com`。
- `DASHSCOPE_API_KEY`：真实模型调用密钥。
- `DASHSCOPE_MODEL`：默认 `qwen-turbo`。
- `DASHSCOPE_BASE_URL`：默认 DashScope OpenAI-compatible endpoint。

## 7. 输出产物

每次运行后生成：

- `outputs/survey.md`：最终调研报告。
- `outputs/mindmap.md`：Mermaid 思维导图。
- `outputs/run_log.json`：完整状态、工具日志、warning 和路径。
- `data/long_term_memory.json`：长期记忆库。

最终提交时，`code/` 文件夹应包含：

```text
code/
  main.py
  requirements.txt
  README.md
  research_team/
    agents.py
    llm.py
    memory.py
    models.py
    tools.py
    workflow.py
```

## 8. 测试计划

- 无 API key 测试：运行 `--mock auto`，确认能生成 `survey.md`、`mindmap.md`、`run_log.json` 和长期记忆。
- arXiv 工具测试：用 `agent memory` 或中文主题检索，确认返回论文；若网络失败，确认 fallback 生效并记录 warning。
- PDF 工具测试：传入 `7-Agent-lab(1).pdf`，确认 `pdf_extract` 能读取文本并写入报告。
- 评分点检查：报告中明确列出 5 个 Agent、Planning/ReAct/Reflection、短期和长期记忆、至少 3 类工具调用。
